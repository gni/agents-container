# clean readme and software documentation guidelines

this skill guides the agent to write clear, direct, and valuable readmes, documentation, and code comments. it focuses on utility and simplicity, eliminating any form of exaggeration, hype, and unnecessary uppercase shouting.

## readability and layout

do not use numbered lists or bullet points to present information. do not write large walls of text that are difficult to scan.

ensure readability by using descriptive lowercase headers for commands and topics.

write short, focused paragraphs or single-line descriptions.

use vertical spacing with empty lines to separate blocks of information.

create simple definition blocks where the name is followed by a colon and a short explanation.

place code blocks directly below their respective descriptions.

## architecture diagrams

visual diagrams like mermaid schemas should be rich and fully detailed. you are encouraged to use emojis, clear block styles, and color codes in mermaid schemas to make them visually informative.

## content guidelines

focus only on true and verified information regarding installation, basic usage, configuration parameters, and architecture. describe the limitations of the system clearly without marketing terms or exaggerated claims. code comments should explain the reasoning behind complex logic rather than repeating what the code does, keeping comments brief and lowercase.
